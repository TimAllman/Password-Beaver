About the Beaver

Password Beaver is a lightweight, easy to use and secure password generator. It creates passwords based upon random selection from the printable characters of the ASCII character set or optionally from an extended ASCII character set. Unwanted characters can be arbitrarily excluded if needed. Optionally, the password is copied to the system clipboard so that it can be pasted where it is needed.

Password Beaver creates a character pool based upon the user's selection of options. The pool of characters starts as the printable characters from either the ASCII or the ISO-8859-1 (the most common extended ASCII) character sets, modified by the userâ€™s selections. It can be modified to omit, request or require characters from several groups. Currently these groups are upper case letters, lower case letters, digits, symbols, and everything else which we refer to somewhat inaccurately as punctuation. Further, an arbitrary set of characters specified by the user can be excluded. You may want to do this to improve readability in case the password needs to be manually copied or perhaps to satisfy a Web site which doesn't accept some characters in its passwords. The length of the password can be set with the minimum length of 8 characters and a maximun of 50.

The generated passwords are intended to be used with a password manager as they are not easy for most people to remember. This makes them harder to crack because a dictionary attack is very unlikely to succeed and a brute force attack will likely take a very long time. Generally the best choice is to pick a long password from a large pool of characters.

Selecting characters

Generally the best choice is to pick a long password from a large pool of characters. If the ASCII character set is chosen (i.e. Extended ASCII is is not chosen) there will be 85 printable characters to select from while selecting Extended ASCII (usually referred to as ISO-8859-1 or Latin 1) will give 155 possible characters.

However, a program or Web site that requires a password may not accept all possible characters or it may set some requirements such as requiring at least one upper case letter, one digit and one punctuation character. Using the top check boxes, it is possible to exclude, allow or require the following groups of characters: upper case letters, lower case letters, digits, symbols and punctuation. These check boxes have three states.

  • If the checkbox is unchecked, that group of characters will be excluded from the character pool.

  • If it is fully checked at least one character from the group will be required.

  • If it is partly checked, characters from that group are requested. They may appear in the password or they may not.

Excluding specific characters

You may exclude certain characters arbitrarily from the character pool. It may be that a Web site does not accept some characters, usually those in the punctuation group. Or, you may wish to avoid characters that are easily confused such as I (upper I) and l (lower L). Check Exclude these characters to turn on this feature and enter the characters in the editor. Unchecking the check box will turn this off even if there are characters in the editor.

Character pool size

This is the current size of the character pool from which passwords are taken and changes as character groups are selected and as some are excluded. This field is read only.

Password length

The length of the password can be changed here. The allowed range is 8 to 50 characters. The default length of 16 is a good choice for most uses.

Copy to clipboard

The default behaviour is to copy the password to the system clipboard as soon as it is generated. Turn this off if you do not want to do this. The text can be selected with the mouse and copied if desired.

Password

This field is read only and contains the last password that was generated. If you don't like it for some reason generate another.

Entropy

The entropy of the password is a measure of how many guesses would be necessary to guess it. Specifically, it is log2(n) where n is the number of possible passwords given the password length and the character pool size. This is described in a Wikipedia article.

Password strength

A good password is hard to guess.
